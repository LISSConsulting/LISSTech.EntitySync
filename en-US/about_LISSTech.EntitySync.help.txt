TOPIC
    about_LISSTech.EntitySync

SHORT DESCRIPTION
    Safe vendor entity synchronization for PowerShell 7.

LONG DESCRIPTION
    LISSTech.EntitySync provides a vendor-neutral synchronization engine with
    ports and adapters for external systems. Adapters support NetSuite
    customers, HaloPSA clients and sites, N-central customers and sites, and
    LCAT customer scopes.

    The module separates discovery, matching, planning, review, and apply.
    Plans are deterministic and can be exported, reviewed, imported, and
    replayed. Write operations require Invoke-EntitySyncPlan -Apply and also
    support PowerShell -WhatIf and -Confirm.

CONFIGURATION
    Configure vendors with Connect-EntitySyncVendor. Values may be passed as
    parameters or supplied from environment variables.

    HaloPSA uses client credentials. Set HALO_BASE_URL, HALO_CLIENT_ID, and
    HALO_CLIENT_SECRET, or pass the matching Connect-EntitySyncVendor
    parameters. The module requests the bearer token from HaloPSA and does not
    require a pre-generated access token.

    N-central uses a username, password, and per-vendor registration token
    (NCENTRAL_BASE_URL, NCENTRAL_USERNAME, NCENTRAL_PASSWORD, NCENTRAL_REGISTRATION_TOKEN),
    or pass the matching Connect-EntitySyncVendor parameters. The module
    derives the OAuth access token from those values.

    LCAT uses a bearer token and only acts as a sync target for N-central
    Customer and Site sources. Set LCAT_BASE_URL and LCAT_BEARER_TOKEN, or
    pass the matching Connect-EntitySyncVendor parameters. To reuse an
    operator-session JWT minted by LISSTech.DeviceAssetOps
    (Get-DeviceAssetOpsAccessToken) without exposing the cleartext token,
    pass it as -LCATSecureBearer (SecureString) instead of
    -LCATBearerToken; the two parameters are mutually exclusive. The bearer
    token is used only for the LCAT authorization header and is omitted from
    Get-EntitySyncConnection, exported plans, dry-run/apply result messages,
    and adapter error paths. LCAT has no customer-scope read endpoint, so
    plans never return target candidates; every approved N-central source
    plans as Create/NoMatch, and site-derived scopes carry the parent
    N-central customer's identifier as ncentral_parent_customer_id.
