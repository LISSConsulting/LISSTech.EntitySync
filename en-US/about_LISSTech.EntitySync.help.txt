TOPIC
    about_LISSTech.EntitySync

SHORT DESCRIPTION
    Safe vendor entity synchronization for PowerShell 7.

LONG DESCRIPTION
    LISSTech.EntitySync provides a vendor-neutral synchronization engine with
    ports and adapters for external systems. Adapters support NetSuite
    customers, HaloPSA clients and sites, N-central customers and sites, and
    AgentController customer scopes.

    The module separates discovery, matching, planning, review, and apply.
    Plans are deterministic and can be exported, reviewed, imported, and
    replayed. Write operations require Invoke-EntitySyncPlan -Apply and also
    support PowerShell -WhatIf and -Confirm.

CONFIGURATION
    Configure vendors with Connect-EntitySyncVendor. Values may be passed as
    parameters or supplied from environment variables.

    For repeated use, run Connect-EntitySyncVendor once with -Profile and
    -SaveProfile. The profile store is local to the current Windows user and
    protects saved settings with DPAPI. Later sessions can run
    Connect-EntitySyncProfile <name> to connect every saved vendor in that
    profile without retyping credentials.

    AgentController profiles store a DeviceAssetOps profile reference, not the
    short-lived AgentController bearer token. On profile connect, EntitySync
    runs Connect-DeviceAssetOps -Profile <name> and then mints a fresh
    session with Get-DeviceAssetOpsAccessToken -AsSession.

    HaloPSA uses client credentials. Set HALO_BASE_URL, HALO_CLIENT_ID, and
    HALO_CLIENT_SECRET, or pass the matching Connect-EntitySyncVendor
    parameters. The module requests the bearer token from HaloPSA and does not
    require a pre-generated access token.

    NetSuite uses OAuth 1.0a credentials for the SuiteQL REST endpoint. Set
    NETSUITE_ACCOUNT_ID, NETSUITE_CONSUMER_KEY, NETSUITE_CONSUMER_SECRET,
    NETSUITE_TOKEN_ID, and NETSUITE_TOKEN_SECRET, or pass the matching
    Connect-EntitySyncVendor parameters. The module signs every SuiteQL
    request with a fresh nonce and HMAC-SHA256 signature per attempt and never
    caches the OAuth header. NetSuite is a source-only adapter in this
    module; target writes go through HaloPSA, N-central, or AgentController.

    N-central uses a User-API token plus the service-organization id for the
    REST API (NCENTRAL_BASE_URL, NCENTRAL_USER_API_TOKEN, NCENTRAL_SERVICE_ORG_ID),
    or pass the matching Connect-EntitySyncVendor parameters. The module
    exchanges the User-API token at /api/auth/authenticate and reuses the
    returned access token for the rest of the REST session. Customer updates
    and organization custom-property writes go through the EI2 SOAP endpoint
    (NCENTRAL_SOAP_USERNAME, NCENTRAL_SOAP_PASSWORD) and are optional unless
    Invoke-EntitySyncPlan or Set-EntitySyncCustomProperty needs them.

    AgentController uses an NSwag-generated client for the ops/PostgREST OpenAPI
    and only acts as a sync target. Build one complete N-central Customer-plus-Site
    snapshot with -SourceEntityType CustomerScope; separate Customer/Site plans
    cannot be applied because omission has authoritative retirement semantics.
    Prefer structured session input from
    LISSTech.DeviceAssetOps: run Get-DeviceAssetOpsAccessToken -AsSession after
    Connect-DeviceAssetOps, then pass that object to Connect-EntitySyncVendor
    -Vendor AgentController -Session. The session supplies both OpsBaseUrl and a
    SecureString operator JWT. EntitySync sends exactly POST
    /rpc/sync_ncentral_customers relative to OpsBaseUrl and does not derive that
    URL from the AgentController API/auth host. Test-EntitySyncConnection uses the
    generated non-mutating has_scope RPC for operator_access:write.

    Manual break-glass mode remains available with LTAC_BASE_URL and
    LTAC_BEARER_TOKEN, or -Url plus -Token/-SecureToken. In manual mode -Url is
    the ops/PostgREST base URL, not the API/auth base URL. The bearer token is
    used only for the AgentController authorization header and is omitted from
    Get-EntitySyncConnection, exported plans, dry-run/apply result messages,
    and adapter error paths. AgentController has no customer-scope read endpoint, so
    plans never return target candidates; every approved N-central source
    plans as Create/NoMatch, and site-derived scopes carry the parent
    N-central customer's identifier as ncentral_parent_customer_id.
